# Bash_utils

## Описание

Учебный проект с реализацией двух классических утилит командной строки из семейства GNU Coreutils — **`cat`** и **`grep`** — на языке **C (стандарт C11)**. Цель — повторить привычное поведение инструментов: построчный вывод файлов с опциями нумерации и отображения непечатаемых символов (`cat`) и поиск по регулярным выражениям POSIX с набором флагов (`grep`).

**Стек:** C11, GCC, GNU Make, Bash (скрипты тестов), clang-format (стиль Google).

## Утилита cat

Копирует содержимое указанных **файлов** на стандартный вывод с учётом опций (часть поведения классического `cat`).

### Поддерживаемые опции

| Опция | Длинная форма (где есть) | Назначение |
|-------|--------------------------|------------|
| `-b` | `--number-nonblank` | Нумеровать только непустые строки |
| `-e` | — | Как `-v`, плюс `$` в конце строки |
| `-E` | — | Только `$` в конце строки (без полного `-v`) |
| `-n` | `--number` | Нумеровать все строки |
| `-s` | `--squeeze-blank` | Сжимать несколько пустых строк подряд |
| `-t` | — | Табы как `^I`, с включением `-v` при `-t` |
| `-T` | — | Только замена табов на `^I` |
| `-v` | — | Отображать непечатаемые символы в «видимом» виде |

Длинные опции задаются через `getopt_long` (см. `cat.c`).

### Пример запуска

```bash
cd src/cat
make
./cat -n test.txt
```

## Утилита grep

Ищет в **файлах** строки, совпадающие с регулярным выражением (POSIX `regcomp` / `regexec`). Поддерживается несколько шаблонов через повтор `-e` и загрузка шаблонов из файла `-f`.

### Основные опции

| Опция | Назначение |
|-------|------------|
| `-e <шаблон>` | Задать шаблон (можно несколько раз, до лимита в коде) |
| `-f <файл>` | Прочитать шаблоны из файла |
| `-i` | Игнорировать регистр |
| `-v` | Инвертировать совпадение (строки **без** совпадения) |
| `-c` | Только количество совпавших строк |
| `-l` | Только имена файлов, где есть совпадение |
| `-n` | Номера строк |
| `-h` | Не выводить имя файла |
| `-s` | Подавлять сообщения об ошибках открытия файла |
| `-o` | Печатать только совпавшую часть строки |

### Пример запуска

```bash
cd src/grep
make
./grep -n "pattern" test_1_grep.txt
```

## Сборка и запуск

### Требования

- Компилятор **GCC** с поддержкой **C11**.
- **GNU Make**.
- Для тестов — **Bash**.
- Для цели `style` — **clang-format** (проверка в стиле Google).
- Для скриптов `test_leak_*.sh` — **valgrind** (если скрипт рассчитан на его использование).

### Сборка отдельной утилиты

**cat:**

```bash
cd src/cat
make        # собрать исполняемый файл cat
```

**grep:**

```bash
cd src/grep
make        # собрать исполняемый файл grep
```

### Очистка и пересборка

```bash
make clean      # удалить объектные файлы и бинарник
make clean && make   # полная пересборка
```

В Makefile также объявлена цель `rebuild`; для гарантированной пересборки удобнее использовать `make clean && make`.

## Тестирование

Функциональные тесты запускаются через **Make**:

```bash
cd src/cat && make test
```

```bash
cd src/grep && make test
```

Скрипты `test_func_cat.sh` и `test_func_grep.sh` сравнивают поведение с эталоном. Скрипты `test_leak_*.sh` предназначены для проверки утечек памяти при установленном valgrind.

---

## Проверка стиля кода

В каждом подпроекте:

```bash
make style
```

Выполняется проверка форматирования: `clang-format -style=Google -n *.c *.h` (без автоматического переписывания файлов).
